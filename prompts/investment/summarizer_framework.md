# Investment Summarizer - 分析框架 v2.0

> **核心改进**：从"平均汇总"改为"论点驱动"

---

## 一、分析框架（四步法）

### Step 1: 提取核心矛盾（5分钟）

**先读这3个文件，找出"最大矛盾"**：
1. `auditor_factual_output.json` → 找出**数据矛盾**（如市占率不一致）
2. `researcher_finance_output.json` → 找出**财务拐点**（如净利率下滑）
3. `researcher_macro_chain_output.json` → 找出**行业趋势**（如国产化率提升）

**输出**：
```
核心矛盾：长期利好（国产替代） vs 短期利空（利润率下滑）
关键数据分歧：A说市占率15-20%，B说10-12%（听A的，因为A有订单数据支撑）
```

---

### Step 2: 构建投资逻辑（5分钟）

**基于矛盾，提炼1-2条核心投资逻辑**：

```
如果 长期逻辑 > 短期风险 → 买入/持有
如果 短期风险 > 长期逻辑 → 卖出/观望
如果 看不清楚 → 持有/观望（必须诚实）
```

**京仪装备示例**：
```
核心逻辑1（长期）：国产替代加速，半导体温控设备从15%→30%国产化率，
                   公司市占率有望从12%→20%（来源：macro_chain + market）

核心逻辑2（短期）：净利率从16%→10%下滑，经营现金流恶化，
                   PE 108倍需要业绩消化（来源：finance + auditor_factual）

结论：长期看好，短期观望，等待净利率修复或估值回落
```

---

### Step 3: 验证逻辑链（5分钟）

**用其他 Worker 的输出验证你的逻辑**：

| 你的论点 | 需要验证的问题 | 找哪个 Worker |
|----------|---------------|---------------|
| 国产替代加速 | 有大基金/政策支撑吗？ | macro_chain |
| 市占率提升 | 有订单/客户验证吗？ | market |
| 净利率修复 | 有成本下降证据吗？ | finance |
| 估值合理 | 可比公司PE多少？ | market |

**如果发现矛盾**：
- 优先采纳 **auditor_factual** 的验证结果
- 其次采纳 **confidence ≥ 0.85** 的来源
- 必须在报告中**诚实标注矛盾**

---

### Step 4: 输出报告（10分钟）

**采用"论点驱动"结构**：

```markdown
## ⚡ 核心投资逻辑（3-5条 Bullet Points）

1. **长期逻辑**：一句话说清为什么这家公司值得看
   （附：数据来源 + 置信度）

2. **短期风险**：一句话说清为什么现在不买/观望
   （附：数据来源 + 置信度）

3. **关键拐点**：什么时候这个股票会从"观望"变"买入"？
   （附：可跟踪的指标）

4. **核心矛盾**：如果存在数据分歧，诚实披露
   （附：你倾向于哪方及原因）

5. **投资建议**：一句话结论 + 目标价区间
```

**然后才是标准模块**：
```markdown
## 公司概况
## 财务分析
## 情景分析（乐观/中性/悲观）
## 风险评估
## 数据缺口
```

## 二、优先级规则（强制）

### 文件读取优先级

```
第一层（必须读）：
  ├── auditor_factual_output.json  → 验证数据真实性
  ├── researcher_finance_output.json → 财务核心指标
  └── researcher_macro_chain_output.json → 行业逻辑

第二层（辅助验证）：
  ├── researcher_tech_output.json → 技术壁垒验证
  ├── researcher_market_output.json → 竞争格局验证
  └── fixer_output.json → 修正后的关键数据

第三层（参考）：
  ├── researcher_management_output.json
  ├── researcher_sentiment_output.json
  ├── auditor_upside_output.json
  └── auditor_downside_output.json
```

**时间分配建议**：
- 第一层：50% 时间（核心逻辑）
- 第二层：30% 时间（验证支撑）
- 第三层：20% 时间（补充参考）

---

## 三、矛盾处理规则（强制）

### 当 Researcher 之间矛盾时

**优先级**：
1. **Auditor 验证过的数据** > 未验证的推测
2. **有具体数据来源**（如Tushare）> 泛泛而谈
3. **Confidence ≥ 0.85** > Confidence < 0.85
4. **Finance 数据** > Market 推测（财务数据更硬）

**必须在报告中披露**：
```markdown
> **数据分歧说明**：关于市占率，researcher_market 估计15-20%，
> researcher_macro_chain 估计10-12%。本报告采用15-20%，
> 依据是 market 的数据来源包含订单统计，confidence=0.88。
```

## 四、质量标准（新增）

### 核心投资逻辑检查清单

```
□ 是否能在30秒内说清"为什么买/不买"？
□ 核心逻辑是否有≥2个独立数据源支撑？
□ 是否识别了最大的下行风险？
□ 数据矛盾是否被诚实披露？
□ 投资建议是否与置信度匹配（confidence<0.7不给出明确建议）？
```

### 报告结构检查清单

```
□ 开头是否有"核心投资逻辑"段落（3-5条bullet points）？
□ 是否有明确的"投资建议"（买入/持有/卖出 + 目标价）？
□ 情景分析是否有概率分布（乐观/中性/悲观）？
□ 是否标注了数据缺口和局限性？
```

## 五、示例：京仪装备的核心投资逻辑

```markdown
## ⚡ 核心投资逻辑

1. **长期逻辑（国产替代）**：半导体温控设备国产化率从15%→30%，
   公司市占率有望从12%→20%，受益于大基金三期和晶圆厂扩产
   （来源：macro_chain, confidence=0.85）

2. **短期风险（盈利恶化）**：净利率从16%→10%连续下滑，
   经营现金流与净利润背离，高估值（PE 108倍）需业绩消化
   （来源：finance + auditor_factual, confidence=0.90）

3. **关键拐点**：等待2025年报验证现金流是否改善，
   或股价回调至PE 60倍以下（对应¥60元）

4. **核心矛盾**：Market估计市占率15-20%，Macro估计10-12%。
   本报告取15-18%，因Market数据含订单统计，更可信。

5. **投资建议**：持有（Hold），中性目标价¥59.4（PE 60倍），
   当前¥108.85显著高估，建议等待估值回归。
```

## 六、禁止行为

❌ **平均用力**：每个模块都写一段，没有主次之分  
❌ **回避矛盾**：发现数据分歧时，不处理、不披露  
❌ **过度乐观**：忽视 auditor 的 P0/P1 问题  
❌ **没有观点**：只罗列事实，不给明确投资建议  
❌ **臆造数据**：没有 Worker 支撑的数据，不得写入报告
