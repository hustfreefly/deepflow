## 关键发现汇总

### 🔴 P0 阻塞问题（必须修复）

#### 1. Research 阶段文件名冲突

| 项目 | 详情 |
|:---|:---|
| **位置** | `task_builder.py` 第 62 行 |
| **问题** | `research_{{expert}}.json` 占位符未替换 |
| **影响** | 3 个 expert 写入同一文件，结果互相覆盖 |
| **修复** | 改为 `research_expert_{i}.json` 或 `research_{expert_name}.json` |

#### 2. Research 阶段占位符未替换

| 项目 | 详情 |
|:---|:---|
| **位置** | `researcher_template.md` + `tasks.json` |
| **问题** | `{{ expert.angle }}` 和 `{{ expert.reason }}` 未提供值 |
| **影响** | 专家角色定义模糊，研究角度不明确 |
| **修复** | Planning 阶段输出 `required_experts` 数组，包含 angle/reason |

### 🟡 P1 重要问题（建议修复）

#### 3. Data Collection 缺乏搜索工具

| 项目 | 详情 |
|:---|:---|
| **位置** | Phase 1: data_collection |
| **问题** | Agent 只有 `web_fetch`，无搜索引擎 |
| **影响** | 无法获取最新行业数据，只能依赖训练数据 |
| **修复** | 提供种子 URL 列表，或增加搜索工具 |

#### 4. Deliver 使用错误 Prompt

| 项目 | 详情 |
|:---|:---|
| **位置** | `task_builder.py` 第 137 行 |
| **问题** | 使用 `architect.md` 而非 `designer.md` |
| **影响** | 输出可能是 JSON 而非 Markdown，格式不匹配 |
| **修复** | 确认使用正确的 deliver prompt |

#### 5. Designer 输入依赖缺失

| 项目 | 详情 |
|:---|:---|
| **位置** | Phase 4: design |
| **问题** | 未明确如何传递 research 输出到 Designer |
| **影响** | Designer 可能无法整合前期研究成果 |
| **修复** | 在 design task 中明确列出需要读取的前置文件 |
